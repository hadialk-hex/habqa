## 💡 1. فكرة المشروع (Project Concept)
**Hubqa (حبقة)** هو منصة سحابية تجارية (SaaS) متكاملة موجهة للسوق العربي (تدعم RTL بالكامل وبخط "Tajawal") لإدارة وتطوير عمليات التواصل التلقائي مع العملاء على منصات التواصل الاجتماعي (Facebook, Instagram, WhatsApp). 

### الميزات الأساسية للمنصة:
1. **صندوق وارد موحد (Unified Inbox)**: تجميع الرسائل والمحادثات من مختلف المنصات والرد عليها من مكان واحد مع دعم التحديث الفوري.
2. **الرد التلقائي على التعليقات (Comment-to-DM)**: (الميزة الأهم تجارياً) عند تعليق العميل بكلمة مفتاحية محددة على منشور، يقوم النظام بالرد عليه علناً بتعليق تلقائي، وإرسال رسالة خاصة فورية له على الخاص (Messenger).
3. **إدارة المشتركين (Subscribers/Contacts Management)**: جمع وتصنيف المشتركين، وإضافة وسوم (Tags) وملاحظات لتسهيل استهدافهم.
4. **الحملات الإعلانية والبث (Broadcasting)**: إمكانية إرسال رسائل جماعية مستهدفة لشرائح معينة من المشتركين وجدولتها.
5. **إدارة القواعد والذكاء الاصطناعي (Rules Engine)**: إنشاء قواعد مخصصة (مثال: إذا احتوى النص على "كم السعر" -> أرسل رد كذا).
6. **إدارة الفريق (Team Management)**: إتاحة دعوة أعضاء فريق جدد وتحديد أدوارهم (Admin, Member, Agent).

## ✅ 2. ما تم إنجازه بالكامل (What Has Been Completed)

تم تحويل المشروع من نموذج أولي بسيط (MVP) ببيانات وهمية وثغرات أمنية إلى **نظام SaaS تجاري متكامل** عبر إكمال 7 مراحل أساسية:

### 🔒 أمن وحماية النظام (Security Hardening)
- تفعيل نظام **JWT Strategy** لإدارة الجلسات بشكل محمي مع تخزين المفاتيح في ملفات الـ Environment.
- تطبيق **JwtAuthGuard** لحماية جميع مسارات لوحة التحكم (`/dashboard/*`) والـ APIs.
- حماية المنفذ الحساس للتسجيل والدخول عبر **Rate Limiting** (15 محاولة كحد أقصى لكل 10 ثوانٍ).
- التحقق الصارم من توقيعات الـ Webhooks الواردة من ميتا عبر تشفير **`X-Hub-Signature-256`**.
- تقييد مشاركة الموارد **CORS** وتمرير هيدرات الحماية (X-Frame-Options, X-Content-Type-Options, XSS Protection).
- **تشفير توكنات فيسبوك الحساسة** المخزنة في قاعدة البيانات باستخدام خوارزمية **AES-256-CBC** لضمان عدم تسريبها.

### 🗄️ قاعدة البيانات وتطويرها (Database Migration)
- تحويل البنية التحتية لقاعدة البيانات بالكامل إلى **PostgreSQL 17** (جاهزة للرفع على سيرفر Oracle ARM).
- إعداد الـ Schema في Prisma لتشمل جداول المشتركين، الحملات، المحادثات، التحقق، السجلات الأمنية (`AuditLog`) ومنع التكرار (`WebhookDeduplication`).
- إضافة فهارس (`@@index`) على الأعمدة الحساسة لتسريع الاستعلامات.

### 🔌 الباكإند والـ APIs الكاملة (Backend & APIs)
يحتوي الباكإند (NestJS 11) على **10 وحدات برمجية كاملة** تسجل **46 مساراً (Endpoint)** تشمل:
- `auth/` — التسجيل، تسجيل الدخول، تعديل الحساب، إعادة تعيين كلمة المرور.
- `channels/` — ربط الصفحات، حفظ التوكنات مشفرة، وتكامل OAuth مع فيسبوك.
- `inbox/` — جلب غرف المحادثات، الرسائل، وإرسال الرسائل.
- `rules/` — إنشاء قواعد الرد، وحساب مقاييس الأداء لكل قاعدة.
- `subscribers/` — البحث المتقدم عن المشتركين وإدارتهم.
- `broadcasts/` — جدولة وبث الحملات وتتبع نسبة وصولها.
- `team/` — دعوة الأعضاء وتوزيع الأدوار الأمنية.
- `webhooks/` — استقبال تعليقات ورسائل فيسبوك وإنستغرام وواتساب.
- `dashboard/` — جلب إحصائيات حقيقية لنسب التفاعل والقنوات النشطة.

### 🤖 أتمتة الويب هوكس والربط الفعلي (Webhooks & Graph API)
- جعل المنصة قادرة على التحدث مع فيسبوك فعلياً عبر استدعاء الـ **Meta Graph API** لإرسال ردود التعليقات والرسائل الخاصة (DMs).
- إدراج آلية تصنيف ذكي للقواعد تفضل القواعد الأكثر دقة والأعلى أولوية.
- منع معالجة الـ Webhooks المكررة عبر جدول مخصص للـ Deduplication.

### 🖥️ الفرونت إند وتكامل البيانات (Frontend Integration)
- ربط جميع صفحات الواجهة (Next.js 16) بالباكإند لجلب البيانات الحقيقية بدلاً من البيانات التجريبية السابقة.
- الحفاظ على واجهات المشتركين والإنستغرام والإعدادات التي قمت ببرمجتها يدوياً وربطها بالـ API.
- بناء قائمة التنقل السريعة المتوافقة مع شاشات الجوال (Mobile Hamburger Menu) في الصفحة الرئيسية.

### 🐳 البنية التحتية والـ Docker (Infrastructure)
- كتابة ملف `docker-compose.yml` جاهز للتشغيل على سيرفر Oracle ARM يتضمن:
  - PostgreSQL 17
  - Redis 7 (اختياري للكاش والـ Queues)
  - NestJS Backend
  - Next.js Frontend
- بناء ملف `Makefile` لتسهيل عمليات الترحيل (`migrate`) وأخذ النسخ الاحتياطية.
- إعداد نظام تسجيل أحداث منظم ومحمي (**Pino JSON structured logging**).

## 🛠️ 4. كيف تشغل المشروع محلياً الآن؟ (How to Run Locally)

لتشغيل المشروع محلياً وتجربته، أمامك خياران:

### الخيار الأول: تشغيل سريع باستخدام SQLite (الأسهل)
1. افتح ملف `backend/prisma/schema.prisma` وغيّر الـ `provider` إلى `sqlite` (السطر 9):
   ```prisma
   datasource db {
     provider = "sqlite"
     url      = env("DATABASE_URL")
   }
   ```
2. تأكد من أن ملف `backend/.env` يحتوي على:
   ```env
   DATABASE_URL="file:./dev.db"
   JWT_SECRET="hubqa-local-dev-secret-2026"
   ALLOWED_ORIGINS="http://localhost:3000"
   APP_SECRET="facebook-app-secret-key"
   ENCRYPTION_KEY="super-secret-encryption-key-32ch"
   ```
3. افتح منفذ الأوامر في مجلد `backend` وشغّل:
   ```bash
   npx prisma generate
   npx prisma db push
   npm run build
   node dist/main.js
   ```

### الخيار الثاني: تشغيل PostgreSQL محلياً عبر Docker (الأفضل لمطابقة الإنتاج)
1. قم بتشغيل PostgreSQL محلياً، ثم عدّل الـ `DATABASE_URL` في `backend/.env` ليشير إلى الـ Database الحقيقية:
   ```env
   DATABASE_URL="postgresql://postgres:postgrespassword@localhost:5432/hubqa?schema=public"
   ```
2. في مجلد `backend` شغّل:
   ```bash
   npx prisma generate
   npx prisma migrate deploy
   npm run build
   node dist/main.js
   ```
